Value props

Dlaczego BoobsOS?

Nie kolejna dystrybucja. Opinionated setup, który po prostu działa.

Gotowy z pudełka

Kompletny zestaw DevOps preinstalowany i skonfigurowany. Kubectl, Terraform, Docker, AWS CLI — wszystko gotowe po pierwszym uruchomieniu. Zero „dzień zero" setupu.

Atomowy i niezniszczalny

Aktualizacje jako całe obrazy OCI. Jedno polecenie bootc rollback cofa system do poprzedniego stanu. System się nie „rozjeżdża" — nigdy.

Reprodukowalny

Cały OS zdefiniowany jak Containerfile. Ten sam obraz na każdej maszynie. Idealny dla zespołów i flot — koniec z „u mnie działa".

Oparty na Fedorze

Świeże jądro Linuksa i toolchainy. Ekosystem RPM + Flatpak (Flathub) + Homebrew. Federacja sprawdzonej bazy z DevOps-owym wierzchem.

Container-native

Docker i Podman obok siebie. Distrobox i Toolbox do izolacji środowisk. Buildah i Skopeo do budowania i inspekcji obrazów. Wszystko gotowe.

GNOME + ciemny motyw

Czysty pulpit Wayland z własnym brandingiem BoobsOS. Nowoczesny, gotowy do pracy. Bez rozpraszaczy, z Flatpak i Flathub po wyjęciu z pudełka.

Architektura

Jak to działa?

BoobsOS to wersjonowany obraz OCI budowany na bazie Fedora bootc.

Fedora bootc baza systemu
→
Containerfile warstwa BoobsOS
→
Build w CI GitHub Actions
→
Obraz OCI ghcr.io / rejestr
→
Twój desktop upgrade / rollback

Aktualizacje atomowe

Zamiast aktualizować setki pakietów osobno, pobierasz nowy obraz całego systemu. Albo wszystko przechodzi poprawnie — albo system pozostaje niezmieniony. Żadnych połowicznych stanów.

Rollback jednym poleceniem

Coś poszło nie tak? sudo bootc rollback i po restarcie jesteś z powrotem na poprzedniej wersji. Bez przeszukiwania logów, bez stresu.

Rebase z Fedory Atomic

Masz już Fedorę Silverblue, Kinoite lub Bazzite? Przejdź na BoobsOS bez reinstalacji — jednym poleceniem bootc switch.

Edycje

Dwie edycje, jedna baza

Ten sam atomowy fundament — wybierz wariant pod swoje zastosowanie.

BoobsOS DevOps

Edycja DevOps/IT — pełny toolchain z pudełka: kontenery, Kubernetes, IaC, chmura, sieć, sekrety. Motyw niebieski (#2563EB). Domyślna edycja.

ghcr.io/filip-zienowicz/boobsos:latest

BoobsOS Game Gaming

Edycja czysto gamingowa (bez narzędzi DevOps): Steam, Lutris, Heroic, ProtonUp-Qt, Discord, OBS oraz gamemode, MangoHud i Gamescope. Motyw czerwony (#DC2626). Dwa warianty GPU:

  • mesa — AMD, Intel, nouveau (otwarte sterowniki z jądra)
  • nvidia — karta NVIDIA (akmod-nvidia preinstalowane przez ublue)

Auto-dobór: zainstaluj wariant mesa, a usługa first-boot wykryje NVIDIA i automatycznie przełączy na wariant nvidia.

ghcr.io/filip-zienowicz/boobsos-game:latest

Przełączaj edycje w dowolnym momencie narzędziem boobsos-edition switch dev lub boobsos-edition switch game — dane w /home zawsze zachowane.
Get started

Instalacja

Trzy ścieżki — wybierz swoją.

1

Rebase z Fedory Atomic

Masz Fedorę Silverblue / Kinoite / Bazzite? To najszybsza droga.

terminal
$ sudo bootc switch ghcr.io/filip-zienowicz/boobsos:latest
# po restarcie — jesteś na BoobsOS
2

Instalacja od zera (ISO)

Pobierz ISO i zainstaluj jak każdy system przez instalator Anaconda (branding BoobsOS). Dostępne obie edycje.

terminal
# edycja DevOps (niebieska) — x86_64
$ curl -LO https://repo.cycx.io/iso/boobsos-dev-44.iso
# edycja Game (czerwona, auto-detekcja GPU)
$ curl -LO https://repo.cycx.io/iso/boobsos-game-44.iso
# zweryfikuj sumę kontrolną SHA-256
$ curl -LO https://repo.cycx.io/iso/SHA256SUMS && sha256sum -c SHA256SUMS
Pobierz ISO DevOps (8.3 GB) Pobierz ISO Game (4.9 GB)
3

Aktualizacja i rollback

Kiedy już masz BoobsOS — trzymaj system aktualny.

terminal
$ sudo bootc upgrade
# cofnij do poprzedniej wersji
$ sudo bootc rollback
4

Instalacja edycji Game

Edycja gamingowa (Steam, Lutris, Heroic, OBS, Discord) bez narzędzi DevOps.

terminal
# AMD / Intel / nouveau (domyślny wariant):
$ sudo bootc switch ghcr.io/filip-zienowicz/boobsos-game:latest
# NVIDIA (akmod-nvidia preinstalowane):
$ sudo bootc switch ghcr.io/filip-zienowicz/boobsos-game-nvidia:latest
# Przełączaj edycje w dowolnym momencie:
$ boobsos-edition switch dev # lub: switch game
5

Obie edycje na jednym dysku

BoobsOS jest obrazowy (bootc/ostree) — „dwie wersje na dysku” robi się inaczej niż klasyczny dual-boot. Dwie drogi:

terminal
# A) PRZEŁĄCZANIE — jedna naraz, wspólny /home (najprostsze)
$ boobsos-edition switch game --apply
$ boobsos-edition switch dev --apply
# B) OBIE NARAZ — wybór w GRUB przy starcie (wspólny /home i /var)
# z zainstalowanej edycji dev twórz drugi deployment:
$ sudo bootc switch ghcr.io/filip-zienowicz/boobsos-game:latest
# przypnij oba, by aktualizacje ich nie usunęły:
$ sudo ostree admin status && sudo ostree admin pin 0 && sudo ostree admin pin 1
# od teraz GRUB pokazuje oba wpisy — wybierasz przy boocie
Wszystkie obrazy (boobsos, boobsos-game, boobsos-game-nvidia) są publiczne na ghcr.io — nie potrzebujesz logowania, żeby je pobrać. ISO: repo.cycx.io/iso.
FAQ

Często zadawane pytania

Czym różni się od zwykłej Fedory?

Zwykła Fedora to baza — instalujesz ją, a potem spędzasz godziny na dostawianiu narzędzi. BoobsOS to opinionated setup: preinstalowany toolchain DevOps, atomowy model bootc (zamiast tradycyjnego rpm-ostree/dnf), własny branding i możliwość rollbacku całego systemu jednym poleceniem.

Czy mogę instalować własne pakiety?

Tak, na kilka sposobów:

  • Flatpak — aplikacje GUI przez Flathub (włączony domyślnie)
  • Homebrew — narzędzia CLI w przestrzeni użytkownika
  • Distrobox / Toolbox — izolowane kontenery z dowolną dystrybucją
  • rpm-ostree — warstwy na systemie (dla zaawansowanych)
Czy stracę dane przy aktualizacji lub rollbacku?

Nie. Katalog domowy (/home) jest trwały i oddzielony od obrazu systemu. Aktualizacje i rollback dotyczą tylko warstwy systemowej — Twoje pliki, ustawienia i dane są bezpieczne.

Na jakim sprzęcie działa?

Aktualnie x86_64. Wsparcie dla aarch64 (ARM) jest planowane. Wymagania sprzętowe są takie same jak dla Fedory — praktycznie każdy współczesny komputer.

Czy to oficjalny produkt Fedory lub Red Hata?

Nie — BoobsOS to niezależna dystrybucja zbudowana na bazie Fedory (model Universal Blue). Nie jest afiliowana z Fedorą Project, Red Hatem ani IBM. To projekt open-source dostępny na GitHub.

Co to w ogóle za nazwa?

Przymrużenie oka — bo dlaczego nie? Produkt jest poważny, nazwa nie musi być. Jeśli to nie Twój klimat, sam Containerfile jest dostępny i możesz zbudować własną dystrybucję pod swoją nazwą.

Jak przełączać się między edycją DevOps a Game?

Jednym poleceniem, bez reinstalacji — /home zostaje nietknięty:

  • boobsos-edition switch game --apply — przełącz na gaming i zrestartuj
  • boobsos-edition switch dev --apply — wróć na DevOps
  • boobsos-edition status — sprawdź aktualną edycję

Edycja Game sama wykrywa GPU NVIDIA i przechodzi na wariant z zamkniętymi sterownikami przy pierwszym boocie. Szczegóły w sekcji Instalacja.

Czy mogę mieć obie edycje na jednym dysku?

Tak. Najprościej — przełączać się (boobsos-edition switch): jedna aktywna naraz, wspólny /home.

Jeśli chcesz obie zainstalowane i wybierać przy starcie w GRUB: z edycji dev odpal sudo bootc switch ghcr.io/filip-zienowicz/boobsos-game:latest, a potem przypnij oba deploymenty, by aktualizacje ich nie usunęły:

  • sudo ostree admin status — zobacz oba (index 0 i 1)
  • sudo ostree admin pin 0 && sudo ostree admin pin 1

Od teraz GRUB pokazuje oba wpisy. Dzielą wspólny /home i /var; każdy aktualizujesz osobno przez sudo bootc upgrade po wejściu w daną edycję.

Skąd pobrać ISO instalacyjne?

Z repo.cycx.io/iso (z brandingiem instalatora BoobsOS):

  • boobsos-dev-44.iso — edycja DevOps (8.3 GB)
  • boobsos-game-44.iso — edycja Game (4.9 GB)
  • SHA256SUMS — weryfikacja: sha256sum -c SHA256SUMS

Nie masz jeszcze systemu? Pobierz ISO i zainstaluj przez instalator Anaconda. Masz już Fedorę Atomic? Użyj bootc switch bez reinstalacji — patrz sekcja Instalacja.
